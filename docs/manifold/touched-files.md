# Touched upstream files (merge-risk note)

> Assignment 7a. Manifold is a fork of Anki, so it must stay mergeable with
> upstream. This note lists every tracked **upstream Anki file** Manifold
> modified, with a one-line merge-difficulty note derived from the actual `git
> diff`, followed by the net-new files Manifold added (which carry no merge risk).
>
> Companions: [`why-rust.md`](why-rust.md), [`alternatives.md`](alternatives.md).

## Modified upstream files

Listed easiest-to-merge first.

- `.dprint.json` (merge risk: low). Adds `.cursor` and `.agents` to the formatter
  `excludes` so stray non-project tooling folders are skipped. Additive entries in
  a list; conflicts only if upstream rewrites the excludes block.
- `README.md` (merge risk: low). Prepends a Manifold overview (the exam, build and
  run, architecture, AGPL credit) above the existing Anki readme, which is kept
  intact below. Additive prepend; conflicts only if upstream rewrites the top.
- `package.json` (merge risk: low). Adds the `@fontsource-variable/fraunces` and
  `@fontsource-variable/nunito` dependencies for the self-hosted UI fonts. Additive
  entries; conflicts only if upstream edits the same dependency block.
- `yarn.lock` (merge risk: low, mechanical). Lockfile entries for those two font
  packages; regenerated by the package manager, so any conflict resolves by
  re-running `yarn`.
- `tools/minilints/src/main.rs` (merge risk: low). Adds `./.agents` to
  `IGNORED_FOLDERS` so the linter skips that stray folder. One additive array
  entry, no logic change.
- `rslib/src/lib.rs` (merge risk: low). One line, `pub mod manifold;`, registering
  the new engine module among the existing `pub mod` declarations. Additive.
- `rslib/proto/src/lib.rs` (merge risk: low). One line,
  `protobuf!(manifold, "manifold");`, registering the new proto in an alphabetized
  macro list. Additive.
- `rslib/proto/python.rs` (merge risk: low). One line, `import anki.manifold_pb2`,
  adding the generated module to the Python proto preamble in an alphabetized
  import list. Additive.
- `qt/aqt/webview.py` (merge risk: low). Adds a `MANIFOLD` value to
  `AnkiWebViewKind` and grants that kind backend API access in `_profileForPage`,
  mirroring the existing `DECK_STATS` entry. Two additive lines.
- `qt/aqt/mediasrv.py` (merge risk: low to moderate). Registers the three Manifold
  SvelteKit pages in `is_sveltekit_page`, and adds backend RPCs to
  `exposed_backend_list`: the Manifold `get_topic_graph` and `build_session_queue`,
  plus the stock scheduler/deck/note RPCs the pages call (`get_deck_id_by_name`,
  `set_current_deck`, `get_queued_cards`, `answer_card`, `grade_now`). Additive
  list entries; conflicts only if upstream restructures those lists.
- `qt/aqt/main.py` (merge risk: moderate). The deepest change. It adds a
  `manifold` main-window state plus a `_manifoldState` handler, points the main
  web view at the `MANIFOLD` kind, redirects `moveToState` to `manifold` on
  collection load, drops Anki's single-letter global shortcuts (they collide with
  the session's A-E keys), adds `hide_native_chrome`, and includes `manifold` in
  `interactiveState`. It touches the window state machine, `MainWebView.__init__`,
  and `setupKeys`, which are all areas upstream edits, so this is the file most
  likely to need a manual merge.

## New files (additive, no merge risk)

These are net-new and live in Manifold-owned paths (or are new files in an
upstream folder), so they do not conflict on merge:

- `proto/anki/manifold.proto`: the `ManifoldService` contract (`GetTopicGraph`
  and the read-only `BuildSessionQueue`, whose `SessionQueueRequest` carries the
  `interleave` flag for the study-feature ablation).
- `rslib/src/manifold/*`: the engine module, made of `mod.rs`, `mastery.rs` (the
  rollup plus the shared lock-state/recall helpers), `session.rs` (the read-only
  DAG-gated session-queue builder with the interleaved or blocked ordering flag),
  `service.rs` (the trait impl), `blueprint.rs`
  - `blueprint.json` (the DAG, weights, and thresholds), and `test.rs` (the Rust
    unit tests).
- `pylib/tests/test_manifold.py`: the Python integration test for
  `get_topic_graph` (a new file under the upstream `pylib/tests` folder; additive).
- `ts/lib/manifold/*`: the proprietary UI library of Svelte components plus the
  scoring, session, graph, and formatting TypeScript and their vitest tests.
- `ts/routes/manifold*`: the home, dashboard, and session SvelteKit routes.
- `ts/tests/e2e/manifold*`: the Playwright end-to-end tests for the three pages.
- `manifold/content/*`: the seed deck, importer, and verifier.
- `manifold/bench/*`: the `get_topic_graph` latency benchmark.
- `docs/manifold/*`: the PRD companions, specs, decision log, and these notes.
